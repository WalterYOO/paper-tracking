# CLAUDE.md

## 仓库说明

本仓库用于跟踪 Arxiv 上的论文进展。各领域论文按子目录分类管理。

## 工作流程

1. 使用 `arxiv-search` skill 获取论文列表
2. 将论文简述更新到对应领域的月度文件中（如 `3dgs/3dgs_2026-05.md`）
   - 文件名格式：`<领域>_<YYYY-MM>.md`
   - 每月一个文件，当月论文写入对应月份的文件
3. 同步创建或更新分类文档（详见"分类文档规范"）
   - 若该月分类文档不存在，从 `category_template.md` 创建
   - 若已存在，将新论文归入对应类别
4. 提交更改并推送到远程仓库

### 论文版本处理

- **不要忽略 v2/v3 等版本更新的论文**。检索结果中带有 v2、v3 或更高版本标识的论文同样需要记录到文件中
- 在论文简述的 arXiv 编号或备注后标注版本号，如 `[2605.17916v3](https://arxiv.org/abs/2605.17916)`
- 若该论文已在本月文件中存在，更新其版本号并补充/更新摘要内容

## 文档格式规范

每个领域的 markdown 文档包含两部分：

### 标题

标题下第一行标注月份，格式为：

```markdown
> YYYY年M月
```

例如：`> 2026年5月`

### 目录（表格）

以表格形式列出所有论文，格式如下：

```markdown
| 日期 | 论文 | arXiv | 类别 | 简述 |
|------|------|-------|------|------|
| 2026-05-18 | [PanoWorld](#锚点) | [2605.17916](https://arxiv.org/abs/2605.17916) | cs.CV | 一句话简述 |
```

锚点格式：小写英文短横线分隔，如 `1-panoworld-a-generative-spatial-world-model...`

### 论文简述

按日期分组，每篇论文格式如下：

```markdown
## YYYY-MM-DD

### 1. 论文标题

- **arXiv**: [编号](https://arxiv.org/abs/编号)
- **作者**: 作者列表
- **类别**: 分类（如 cs.CV）
- **备注**: （可选，如录用信息）

**摘要**: 中文摘要，简述研究背景、方法、结果。
```

多篇论文之间用 `---` 分隔。

详细格式示例请参考 `tracking_sample.md`。

### 亮点论文标记

表格中若论文被顶会/顶刊录用，在论文名前加 `★ ` 并对论文名做**加粗**，格式为：

```markdown
| 2026-04-30 | ★ **[FieryGS](#锚点)** | [arxiv](链接) | cs.GR | 简述（ICLR'26） |
```

**顶会/顶刊判断标准**（仅在录用信息明确标注时标记）：

- 计算机视觉：CVPR、ICCV、ECCV
- 机器学习：NeurIPS、ICML、ICLR
- 图形学：SIGGRAPH、SIGGRAPH Asia、Eurographics
- 机器人：RSS、ICRA、IROS
- 多媒体：ACM MM
- 设计自动化：DAC、ISCA
- 其他公认的 A\*\* 级别会议/期刊

Highlight、Oral、Spotlight 等子类别也标记。Workshop 若属于上述会议的 Workshop 可标记。

在目录表格上方添加说明行：
```markdown
> 注：★ 表示被顶会录用。
```

### 排序规则

目录表格和正文日期区块均按**日期从近到远**（降序）排列。

#### 实操要求

1. **确定插入位置**：新检索到的论文需要找到其日期在现有内容中的正确边界。
   - 新日期比所有现有日期更新 → 插入到表格和正文的最前面（最旧日期之前）
   - 新日期比所有现有日期旧 → 追加到表格末尾和正文最后
   - 新日期在现有日期之间 → 插入到对应区间
   - 与已有日期相同 → 合并到该日期的现有区块中

2. **表格插入**：新行应插入到分隔线（`|------|`）之后，确保从上到下日期递减。
   - 表格行与日期区块一一对应，顺序必须一致

3. **正文插入**：新的 `## YYYY-MM-DD` 区块及其内容应插入到正确的位置，确保日期区块从上到下递减。

4. **日检对比**：完成插入后，快速用 `grep -n "^## YYYY" 文件路径` 或 `grep -n "^| 2026" 文件路径` 确认所有日期是否从近到远排列。

#### 示例

假设文档当前已有 2026-05-15 和 2026-05-10 的论文，新检索到 5 月 12 日的论文：

**错误做法** — 放在最前面或最后面：
```
| 2026-05-12 | ... |  ← 放在最前面（错）
| 2026-05-15 | ... |
| 2026-05-10 | ... |
```

**正确做法** — 插在 5-15 之后、5-10 之前：
```
| 2026-05-15 | ... |
| 2026-05-12 | ... |  ← 正确位置
| 2026-05-10 | ... |
```

#### 另一个插入示例（以实际文件结构为例）

假设文档当前正文末尾如下（已有 5-13 到 5-11）：
```
## 2026-05-13
### 1. PanoPlane
...
### 6. SCOUP
---

## 2026-05-12
### 1. XFreq-GS
...
---

## 2026-05-11
### 1. PG-3DGS
...
### 6. VEGA
---
```

新检索到 2026-05-09 的论文，正确插入位置：

**表格中** — 在 5-10 行之后、5-08 行之前的位置（若无 5-08 则在 5-10 之后直接插入）：
```
| 2026-05-10 | [ConFixGS](...) | ... |
| 2026-05-09 | [新论文A](...) | ... |  ← 插入
| 2026-05-09 | [新论文B](...) | ... |  ← 插入
| 2026-05-08 | [GD-DIFF](...) | ... |
```

**正文中** — 在 `## 2026-05-10` 区块结束后、`## 2026-05-08` 区块之前插入新区块：
```
## 2026-05-10
### 1. ConFixGS
...
---

## 2026-05-09                        ← 新日期区块插在这里
### 1. 新论文A
...
### 2. 新论文B
---

## 2026-05-08
### 1. GD-DIFF
...
```

关键：表格行和正文区块必须同步插入，且插入后表格顺序与正文顺序完全一致。

#### 批量插入多日论文

如果一次性检索了多天（如 5-10 到 5-1）的论文：
- 先整体判断这批论文与现有日期的相对位置
- 再将整批内容按内部已排好序的顺序，作为一个整体插入到正确边界
- 表格和正文同步插入，不要只改表格不改正文或反之
- 插入完成后用 `grep -n "^## 2026-05-" 文件路径` 确认正文日期顺序

## 分类文档规范

月度论文收集完成后，需要编写分类文档，按研究方向对论文进行归类。

### 工作流程

1. 复制 `category_template.md` 为 `<领域>_category_YYYY-MM.md`，放置在同领域子目录中
2. 将标题和说明中的 `YYYY-MM` 替换为对应月份
3. 从月度文件中读取论文，按研究方向填入对应类别的表格
4. 更新底部的统计概览

### 文档格式

- 表格列：`| 日期 | 论文 | arXiv | 关键词 |`
- 每个分类包含：表格 + 概要段落
- 表格行按日期从近到远（降序）排列

### 分类规则

- 论文按最匹配的研究方向归类，一篇论文只归一个主类别
- 若论文跨多个领域，归入最核心的研究方向
- 若某分类本月无论文，删除该分类（不保留空分类）
- 分类编号使用中文数字（一、二、三...），删除后重新编号

### 锚点要求

- 标题后添加显式 HTML 锚点 `{#id}`，格式如 `## 一、核心算法与优化 {#cat-core-opt}`
- 目录链接与标题锚点保持一致
- 论文链接指向月度文件中的对应锚点

### 亮点论文标记

与月度文件相同，顶会/顶刊录用论文格式为 `★ **[论文名](#锚点)**`，关键词列末尾标注会议信息（如 `CVPR'26`、`SIGGRAPH'26`）。

### 统计概览

文档底部统计表格只列出实际有论文的类别，论文数按模板中实际数量填写。
